HO vs SL — Extraction Philosophy
HO — DOCX Handout (expand)
  • Main Points → chapter intro prose
  • Talking Points → full prose paragraphs
  • Analogy → 2–3 prose sentences in callout
  • Worked Example → full Before/After + explanation
  • Watch Outs → prominent warning callout boxes
  • CTA → full 2–3 sentences on dedicated page
SL — MARP Slides (compress)
  • Main Points → headline bullet (≤10 words)
  • Talking Points → 2–3 distilled bullets (≤12 words)
  • Analogy → 3-bullet comparison, one line each
  • Worked Example → Before/After one-liner contrast
  • Watch Outs → excluded from slides entirely
  • CTA → 2 lines max

Golden Rule

Fix problems upstream, not downstream.

If HO and SL share the same issue, fix the part draft — not the format outputs.
If multiple parts share the same issue, fix the MD volume run — not individual parts.
Common fixes
Problem Fix
Output is too generic Enrich MD — add concrete examples to EXAMPLES_OR_ANALOGIES. Never fix in format prompts.
Wrong tone Add phrasing examples to TONE field: "Write like this: [example]. Not like this: [bad example]."
Tone wrong for one part only Set TONE_OVERRIDE in that part's block. Re-run only that part's format outputs.
DOCX reads like a bullet dump Re-run HO: "REMINDER: convert all TALKING POINTS to prose paragraphs."
MARP bullets too long Re-run SL: "REMINDER: max 12 words per bullet. Max 3 bullets per content slide."
MARP slides feel like a handout Re-run SL: "REMINDER: distil, do not copy. Each bullet is the one thing to remember."
One part repeats another Check VOLUME ORIENTATION CONCEPT MAP. Re-run MD with a correction block.
Volume scope too broad Edit the CP output directly. Split into two volumes — never patch in MD.
